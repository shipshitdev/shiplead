# USER-PREFERENCES

## Critical Rules

- Direct, no fluff. Implement when intent is clear.
- Read-only commands do not need approval. Run them.
- Before writing new code, find at least three real examples in the codebase or adjacent local ecosystem and match the dominant pattern.
- Do not ask Vincent to write code.
- Avoid destructive commands unless Vincent explicitly asks.

## Quality Bar

- Favor consistency over novelty
- Add guardrails and tests when they improve autonomy
- State missing context plainly instead of papering over it

## Operating Context

- Vincent is cost-aware; do not burn expensive models or long workflows on trivial work
- Vincent thinks at founder and CTO level; explain tradeoffs in terms of speed, risk, and leverage
